# Цель работы

Освоить сохдание и оформление таблиц в $LaTex$ с использованием окружения `tabular`, изучить выравниевание столбцов, поведение таблицы при недостаточном или избыточном количестве ячеек в строке, а также примеры команды `\multicolumn` и возможностей пакета `array` для настройки формата столбцов. 

# Задание


1. Использовать простой пример таблицы и начать экспериментировать с табличной разметкой.

2. Проверить разные выравнивания столбцов с помощью `l`, `c`, `r`.

3. Проверить, что происходит, если в строке таблицы **меньше элементов**, чем описано в преамбуле.

4. Проверить, что происходит, если в строке таблицы **больше элементов**, чем описано в преамбуле.

5. Поэкспериментировать с `\multicolumn` для объединения ячеек по горизонтали и с настройкой форматирования столбцов через пакет `array`.

# Выполнение лабораторной работы

## 1) Создание базовой таблицы

Была создана таблица в окружении `tabular` с тремя столбцами. В качестве данных использованы простые текстовые значения (животное, еда, размер). Это позволило проверить базовый синтаксис: разделение ячеек символом `&` и завершение строки `\\`.

## 2) Эксперименты с выравниванием столбцов

Были собраны три варианта одной и той же таблицы:

* `lll` — выравнивание по левому краю;

* `ccc` — выравнивание по центру;

* `rrr` — выравнивание по правому краю.

Сравнение результатов показывает, что типы столбцов напрямую влияют на расположение содержимого в ячейках, но структура таблицы не меняется.

## 3) Рассмотрение случая со слишком малым кол-вом элементов в таблице

Для проверки поведения LaTeX при недостатке ячеек была создана строка, содержащая только (2) элемента при объявленных (3) столбцах. В результате третья ячейка в этой строке осталась пустой, ошибок компиляции не возникло.

  

Вывод: если элементов меньше, чем требуется, LaTeX не “достраивает” данные автоматически, а оставляет недостающие ячейки пустыми.

## 4) Рассмотрение случая со слишком большим кол-вом элементов в таблице

Был подготовлен пример строки с (4) ячейками при объявленных (3) столбцах. Такой код в обычном виде приводит к ошибке выравнивания (типичное сообщение: *Extra alignment tab has been changed to* …).

Чтобы документ оставался компилируемым, демонстрационный фрагмент “ошибочного” кода был размещён внутри окружения `verbatim`. Это позволило показать пример и пояснить проблему, не ломая сборку PDF.

Вывод: избыток `&` (лишняя ячейка) приводит к ошибке; исправление — либо убрать лишнюю ячейку, либо объявить больше столбцов в преамбуле.

## 5) Эксперименты с `\multcolumn` и пакетом `array`

### 5.1) Объединение ячеек по горизонтали

С помощью `\multicolumn{2}{c}{...}` были объединены (2) последних столбца в одну ячейку, что демонстрирует “склейку” по горизонтали и возможность задавать выравнивание (`c`) для объединённой области.

### 5.2) Управление форматированием отдельных ячеек и столбцов

С использованием пакета `array` была настроена преамбула таблицы вида `>{\itshape}l<{:}ll`, что привело к:

* курсивному оформлению первого столбца;

* добавлению символа `:` после содержимого первого столбца.

Чтобы заголовок первого столбца не наследовал курсив и двоеточие, применена конструкция `\multicolumn{1}{l}{...}`, которая локально “отменяет” заданные модификаторы форматирования.

Дополнительно для более аккуратных горизонтальных линий использован пакет `booktabs`, дающий типографический корректный вид таблицы.

# Результаты 

1. Создан документ LaTeX, демонстрирующий базовую табличную разметку.

2. Проверены режимы выравнивания столбцов `l`, `c`, `r`.

3. Установлено:

- при недостатке ячеек строка компилируется, недостающие ячейки остаются пустыми;

- при избытке ячеек возникает ошибка выравнивания.

4. Освоены приёмы:

- объединение ячеек по горизонтали через `\multicolumn`;

- настройка формата столбцов через пакет `array`;

- использование `booktabs` для качественных линий в таблицах.

# Выводы

В ходе выполнения лабораторной работы были изучены основные инструменты создания таблиц в LaTeX. Практика показала, что корректность таблицы строго зависит от согласованности количества столбцов и числа ячеек в строках. Команда `\multicolumn` и пакет `array` позволяют гибко управлять структурой и форматированием таблиц, а пакет `booktabs` улучшает типографское качество оформления.
